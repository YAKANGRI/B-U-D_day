## 1. 편집
#### Data Transfer [데이터 전송] 
한 오브젝트의 데이터를 다른 오브젝트로 복사합니다.
- 1. 로우폴리곤 모델에 하이폴리곤의 부드러운 노멀 (normal) 정보를 입힐 때, 복잡한 옷 모델링 몸체에 웨이트 값을 복사할 때 또는 서로 다른 메쉬 간에 버텍스 그룹, UV 맵 등을 공유할 때 사용
  2. 메쉬 구조가 달라도 데이터를 옳길 수 있다는 것이 장점

  * 몸체 모델링에는 웨이트 값이 잘 설정되어 있지만, 옷에는 설정이 되어있지 않을 때 이 기능을 사용하여 크게 수정없이 사용 가능

  * 각진 상자에 복잡하고 부드러운 구체의 노멀 값을 각진 상자에 넣어 빛을 받았을때 부드럽게 보이도록 만들 수 있음

#### Mesh Cache [메쉬 캐시] 
외부 소프트웨어(Houdini, Maya 등)에서 계산된 애니메이션 데이터를 불러옴
- 1. .pc2, .mdd (mesh Cache)나 .abc (Alembic, Sequence Cache) 파일을 읽어와 블랜더 내에서 재생

   2. 블랜더 내에서 수정은 어렵지만, 매우 무거운 시뮬레이션 데이터를 안정적으로 보여줌
   * 블랜더 밖에 쓴 적이 없기 떄문에 이 기능에 대해선 이해가 가질 않아 나중에 실험해볼 필요가 있음
 
#### Mesh Sequence Cache [메쉬 스퀀스 캐시] 
Alembic (.abc) 파일 형식을 사용하여 복잡한 애니메이션이나 시뮬레이션 데이터를 불러옴
- 1. 유체 시뮬레이션이나 캐릭터 리깅 애니메이션처럼 프레임마다 메쉬의 구조가 변하는 대용량 데이터를 효율적으로 관리하고 재생함

  * 블랜더 밖에 쓴 적이 없기 떄문에 이 기능에 대해선 이해가 가질 않아 나중에 실험해볼 필요가 있음

#### UVProject [UV 투영]
마치 빔 프로젝터처럼 특정 오브젝트 (주로 Empty, Camera) 기준으로 이미지나 텍스처를 메쉬 위에 쏘는 방식 
- 1. 복잡한 UV 언랩(Unwrap) 과정 없이 특정 방향에서 로고를 입히거나 데칼(Decal) 작업을 할 떄 매우 편리함

  * 보이는 대로 로고나 그림을 텍스처를 입히고 싶을 떄 쓰는 기능. 쓸 일은 없을것 처럼 보임 (아직)

#### VertexWeightEdit [버텍스 웨이트 편집]
기존에 설정된 버텍스 그룹의 가중치 (Weight) 값을 수정
- 1. 커스텀 커브를 이용해 웨이트값의 분포를 조절하거나, 특정 범위 이하의 웨이트를 가진 버텍스 그룹에서 제거하는 등 정교한 웨이트 관리가 필요 할 때 사용함

  * 어디를 얼마나 변형시킬지 수치로 정해주는 조절기 (적용하면, 버텍스 그룹 값이 수정되거나 조절됨)

#### VertexWeightMix [버텍스 웨이트 믹스] 
두 개의 버텍스 그룹을 서로 섞거나(Mix), 한 그룹의 값을 다른 그룹에 더하고 빼는 등의 연산을 수행 
- 1. 복잡한 리깅 시스템에서 여러 부위의 웨이트 영향을 합치거나 조절할 때 필수적인 도구. add, Multiply, Difference 등 겹치는 부분을 남기거나 빼는 등 여러가지 기능이 들어있음 
 
  * "A 그룹이랑 B 그룹을 더하라" 또 "A에서 C 영역만큼 빼라" 같은 눈에 보이는 계산을 담당함 다양한 기능으로 그냥 합치기 기능과 전혀 다름
 
#### VertexWeightProximity (버텍스 웨이트 근접)
다른 오브젝트와의 거리를 기반으로 버텍스 웨이트를 자동 생성함 
- 1. 특정 물체가 가까이 올 때만 풀이 흔들리게 하거나, 물체가 닿는 부분만 텍스처가 변하게 하는 등의 인터렉티브한 효과를 만들 때 사용함.
 
  * 유니티로 치자면 콜라이더 박스(히트 박스) 과 오브젝트를 타겟을 이용해 C# 으로 만든 것을 시각적으로 표현한것. 자주 쓰게 될거 같음.
